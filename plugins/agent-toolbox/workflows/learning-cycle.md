# Learning Cycle

Use this workflow when the user wants to learn a topic from the ground up and build durable
understanding.

## 1. Learning Intake

Owner: Learning Coach

- Identify the topic.
- Briefly diagnose the learner's current level, motivation, and desired depth.
- Ask at most one or two short questions if the starting point is unclear.
- Choose whether Research Analyst is needed for current facts or canonical references.

Output:

- Learning goal.
- Starting level.
- Depth target.
- Research needs, if any.

## 2. Memory Load

Owner: Learning Coach

- Read relevant user preference memory and learning-coach memory.
- Check for recurring misconceptions, preferred style, or prior learning notes.
- Treat current user instructions as higher priority than memory.

Output:

- Relevant learning context.
- Memory conflicts or stale notes.

## 3. First-Principles Lesson

Owner: Learning Coach

- Start with the one-sentence version.
- Explain the motivation and the problem the idea solves.
- Use the simplest useful example before formal definitions.
- Build the durable mental model.

Output:

- Core explanation.
- Worked example.
- Mental model.

## 4. Real-World Layers

Owner: Learning Coach

- Add complexity one layer at a time.
- Explain why each layer exists.
- Show common failure modes and beginner traps.
- Use diagrams, code, math, or stories when they improve understanding.

Output:

- Production or real-world version.
- Failure modes.
- Misconception warnings.

## 5. Active Check

Owner: Learning Coach

- Ask the learner to predict, explain, compare, or apply the concept.
- Avoid relying on "does that make sense?"
- If the learner struggles, repair the misconception with a different representation.

Output:

- Check question.
- Learner answer or pending exercise.
- Misconception notes.

## 6. Practice

Owner: Learning Coach

- Give one focused exercise, experiment, or mini-project.
- Keep the exercise small enough to complete soon.
- Offer a review path for the learner's attempt.

Output:

- Exercise.
- Expected learning outcome.
- Optional answer key or review criteria.

## 7. Memory Update

Owner: Learning Coach

- Save durable preferences or recurring misconceptions only when useful for future learning.
- Do not store private personal details or hidden reasoning.
- Follow `memory/schema.yaml`.

Output:

- Memory entries added, updated, or intentionally skipped.
